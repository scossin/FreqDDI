Le programme a été développé pour dénombrer des interactions médicamenteuses à partir de données de délivrance issues du dossier pharmaceutique. 
Les fichiers en entrées sont au format CSV. Il est nécessaire d'adapter le programme avec un autre jeu de données. 
Un fichier fictif est disponible dans le répertoire data2
Le dossier index doit être créé

Les étapes du programmes sont les suivantes : 
	- indexer les "ordonnances" délivrées. Pour simplifier, on peut fournir en entrées une liste de codes CIP "inclus", c'est à dire que si le code CIP est inconnu (ex : pas une spécialité pharmaceutique),
	il ne sera pas indexé. 
	- dénombrer la fréquence des ordonnances, pour cela :
		- un fichier contenant pour chaque molécule (présente obligatoirement dans le fichier interaction) : molécule - codes CIP
		- un fichier d'interaction : molécule1 - molécule2 ; signifiant que molécule1 interagit avec molécule2. 
		fonctionnement : Le programme cherche dans l'index Lucene : CIP(molécule1) intersection CIP(molécule2)

1) IM/interaction/RepertoireMedicament/
	Mettre à jour les fichiers du répertoire du médicament
	Mettre à jour les alignements si besoin entre le thésaurus et le répertoire du médicament
	Faire tourner chargement.R
	puis dciCIP.R

2) IM/interaction/
	Créer un nouveau dossier pour y mettre les fichiers concernant les interactions
	listeInteractions.R en premier puis
	CIP_inclus_exclus.R

3) Indexer les fichiers avec le programme JAVA. 
4) Dénombrer les interactions
	- denombre : fournit le nombre N d'ordonnances contenant l'interaction
	- denombredetails : fournit le détail de chaque ordonnance. Si N ordonnances interagissent, on aura N lignes. Chaque ligne : molécule1, molécule2, numordonnance, age, Nmedocs sur l'ordonnance

5) Le dossier output sort les résultats de l'analyse
